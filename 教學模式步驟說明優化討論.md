# 教學模式步驟說明優化討論

本文件對照《血球分類軟體介面說明.txt》的**設計目的與意義**，逐步驟討論教學文案應納入的重點，供您確認或修改後再寫回 `assets/js/tutorial.js`。

---

## 一、介面說明中的設計目的摘要（對照用）

| 代號 | 問題／目的 | 對應介面 |
|------|------------|----------|
| **方案總述** | 集中化版本管理、突破硬體限制、資訊串接（FHIR／雲端） | 整體 |
| **T1** | 外部資訊斷裂與工作流中斷 → 減少跨系統查詢、避免無效作業 | 檢體管理、檢體總覽 |
| **T2** | 實體位置與資源追蹤缺失 → 發現異常時能快速調度實體覆核 | 檢體歸位、操作機台 |
| **T3** | 檢索效率與清單架構缺陷 → 高檢體量下降低認知負荷 | 篩選、分頁、排序 |
| **T4** | 系統協作與容錯性不足 → 避免多人同時作業衝突、重工 | 鎖定狀態、已完成標記 |
| **T5** | 介面可讀性與人因工程 → 減輕顯微鏡與螢幕切換的視覺疲勞 | 字體縮放 |
| **影像 T1** | 外部資訊斷裂與任務指引缺失 → 判讀資訊一目了然、無需切換視窗 | 側邊儀表板 |
| **影像 T2** | AI 辨識準確度與效能 → 信心指數與建議輔助重分類決策 | AI 建議、右鍵選單 |
| **影像 T3** | 介面資訊可視性不足 → 高風險細胞不被遺漏 | 異常置頂、紅色邊框 |
| **影像 T4** | 系統協作與資料邏輯 → 自動跳下一筆、排除鎖定/需人工鏡檢 | 智慧導航 |
| **細胞 T1** | 單手操作與人因工程 → 減少手部切換、減輕負擔 | 單手框選、右鍵＋左鍵 |
| **細胞 T2** | 操作精準度與介面佈局 → 落點清晰、選單分層易點擊 | 群組邊框、右鍵選單 |
| **細胞 T4** | 系統邏輯與回饋滯後 → 達留單標準時即時通知 | 即時標籤、閾值變色 |
| **報告 T1** | 異常值視覺警示與防錯 → 留單標準時主動攔截 | 紅／黃／綠橫幅 |
| **報告 T2** | 決策支援資訊斷裂與回饋滯後 → 同一畫面比對四種數據 | 整合比較表、CBC、病患卡 |
| **報告 T3** | 介面可讀性與無效點擊 → 字體、排版、免除強制無效點擊 | 字體與排版 |
| **報告 T4** | 流程不可逆與修正彈性 → 誤簽可撤回 | 撤回功能 |

## 二、檢體管理頁（SPECIMEN_STEPS）— 逐步驟討論

### 步驟 0：歡迎使用

| 項目 | 內容 |
|------|------|
| **現有** | 本教學將帶您走過完整流程：檢體管理 → 影像檢視與細胞編輯 → 報告核發。請依序操作，或隨時可跳過。 |
| **設計目的可納入** | 無直接對應單一 T；可簡短帶出「本系統為網頁版，集中更新、易與 LIS 串接」，讓使用者理解為何是這套流程。 |
| **建議優化重點** | （1）是否要加一句「目的：讓您快速掌握從檢體清單到簽核的完整工作流」？（2）是否提及「醫檢師」角色以增加代入感？ |

---

### 步驟 1：檢體管理 — 整體介面

| 項目 | 內容 |
|------|------|
| **現有** | 本頁分為左側與右側：左側為篩選區與檢體列表，右側為檢體總覽。系統預設篩選當日檢體避免資料混雜；您可切換數位閱片／實體作業模式，並自訂篩選條件。以下依左上→左下→右上→右下介紹各區塊。 |
| **設計目的可納入** | **T1**：右側檢體總覽整合 LIS 相關資訊，減少跨系統查詢。**T3**：篩選、模式分頁正是為「高檢體量下降低搜尋負荷、維持清單直覺」而設計。 |
| **建議優化重點** | （1）明確寫出「右側檢體總覽可一次看到流式計數、AI 預分類、前次報告等，減少在 LIS／其他系統間切換」。（2）「數位閱片／實體作業」可點出是為了把「要電腦判讀」與「要實體拉片／血小板確認」分開，避免做錯類型的工作。 |

---

### 步驟 2：左上 — 篩選與模式

| 項目 | 內容 |
|------|------|
| **現有** | 數位閱片／實體作業：切換工作模式。數位閱片預設顯示待閱片檢體；實體作業預設顯示需推片或實體採血管確認的檢體。篩選：點擊可展開檢體狀態勾選。檢驗日期、送檢單位、機台、檢體搜尋：可依…篩選。 |
| **設計目的可納入** | **T3**：工作流分頁與多維度檢索（日期、單位、機台、ID）是為「檢索效率與清單架構」服務，降低認知負荷。 |
| **建議優化重點** | （1）加一句「在高檢體量時，先用模式與篩選縮小範圍，可更快找到要處理的檢體」。（2）「僅顯示完全相符項」可說明為「精確鎖定單一檢體時使用」。 |

---

### 步驟 3：左下 — 檢體列表與狀態膠囊

| 項目 | 內容 |
|------|------|
| **現有** | 檢體列表顯示每筆檢體的資訊，包含狀態膠囊；並列出各膠囊顏色與意義（PLT Check、AI Alert、Follow-up、Digital Review、Verified、Locked）。 |
| **設計目的可納入** | **T1**：視覺化狀態膠囊讓醫檢師「直接知道該筆檢體需要做哪些處置」，減少在不需要數位閱片或必須人工鏡檢的檢體上做無效作業。**T2**：列表中的「操作機台」「檢體歸位」欄位是為「發現異常需回頭複檢時，快速找到實體玻片／採血管位置」。**T4**：Locked / Verified 標示是為「避免多人同時編輯同一筆、或誤點已完成的檢體」。 |
| **建議優化重點** | （1）在介紹膠囊前，先寫一句「狀態膠囊的設計目的，是讓您一眼看出每筆檢體該做數位閱片、血小板確認還是實體鏡檢，避免做錯或重複處理」。（2）補充「操作機台」與「檢體歸位」的用途：需要實體覆核時，可依此找到玻片匣或採血管位置。（3）說明 Locked（斜紋＋鎖頭）表示他人編輯中、請勿點選；Verified（綠色打勾）表示已完成，避免重工。 |

---

### 步驟 4：右上 — 檢體總覽標題

| 項目 | 內容 |
|------|------|
| **現有** | 右側上方顯示檢體總覽標題與目前選取檢體的 ID。點選左側列表任一列後，此區會對應顯示該筆檢體。 |
| **設計目的可納入** | **T1**：整合資訊側邊欄的目的就是「點擊列表後在右側一次看到病患基本資訊與數據對照」，減少跨系統查詢。 |
| **建議優化重點** | （1）可改為「點選左側任一檢體後，右側會顯示該筆的總覽，不需切到其他系統即可看到病患與數據」，強調「同一畫面、不切換」的設計意義。 |

---

### 步驟 5：右下 — 檢體資訊與分析

| 項目 | 內容 |
|------|------|
| **現有** | 右側下方顯示選取檢體的檢體資訊（病歷號、姓名、科別等）與分析與歷史報告（流式計數、AI、前次報告）。未選取時會提示「請從左側列表點選一筆檢體，或雙擊進入閱片」。 |
| **設計目的可納入** | **T1**：並列對照流式計數、AI 預分類、前次報告，異常數值以紅色粗體警示，對應「減少跨系統查詢、輔助判讀」。 |
| **建議優化重點** | （1）明確寫「異常數值會以紅色粗體標示，達留單標準的項目會特別突出，方便您判斷是否需要進一步處置」。（2）可簡短提及「數據來源對應 LIS／XN-9000、AI 與前次報告」，讓醫檢師理解資訊整合的意義。 |

---

### 步驟 6：進入下一階段（檢體列表 / 雙擊或進入閱片）

| 項目 | 內容 |
|------|------|
| **現有** | 請雙擊任一檢體列，或先點選一列後點該列右側的「進入閱片」按鈕，即可進入影像檢視與細胞編輯頁面，教學將在該頁繼續。 |
| **設計目的可納入** | 銜接下一階段；可順帶提醒「若該列顯示 Locked，表示他人使用中，請選其他檢體」。 |
| **建議優化重點** | （1）可加一句「進入閱片後，左側會延續顯示本檢體資訊與數據，無需另開視窗」。（2）選填：提醒「已完成的檢體（綠色打勾）不需再點入」。 |

## 三、影像檢視與細胞編輯頁（IMAGE_STEPS）— 逐步驟討論

### 步驟 0：影像檢視 — 整體介面

| 項目 | 內容 |
|------|------|
| **現有** | 本頁分為左側與右側：左側為檢體資訊、狀態、搜尋與分析表；右側為細胞圖卡與工具列。以下依左上→左下→右上→右下介紹。 |
| **設計目的可納入** | **影像 T1**：整合式側邊儀表板的目的就是「檢視細胞時同步看到病患資料、Flow、歷史與狀態」，消除視窗切換的認知負擔。 |
| **建議優化重點** | （1）開頭加「在這裡您可以在同一畫面完成閱片與數據對照，不需在 LIS、顯微鏡與本系統間來回切換」。（2）可簡短點出「右側細胞區會依類別分組，異常細胞會置頂並以紅色標示，避免漏看」。 |

---

### 步驟 1：左上 — 導航與狀態

| 項目 | 內容 |
|------|------|
| **現有** | 返回列表、檢體 ID、狀態膠囊、Add Flag、搜尋、上一筆／下一筆。 |
| **設計目的可納入** | **影像 T4**：智慧導航（上一筆／下一筆、待辦數量）是為「完成當前檢體後自動跳下一筆待數位閱片，並排除鎖定與需人工鏡檢者」，提升效率並防止重複作業。**狀態標記**：醫檢師可手動新增標記（如發現 AI 未辨識到的異常）。 |
| **建議優化重點** | （1）說明「完成並儲存報告後，系統會自動跳到下一筆待數位閱片的檢體，且會略過正在被他人編輯的檢體」。（2）「Add Flag」可說明為「當您發現 AI 未標記的異常時，可手動為此檢體加註狀態，方便後續追蹤」。 |

---

### 步驟 2：左下 — 檢體資訊與分析

| 項目 | 內容 |
|------|------|
| **現有** | 檢體資訊可展開／收合；分析與歷史報告為流式、AI、前次報告，異常值紅色標示。異常標準參照林口長庚醫院留單標準。 |
| **設計目的可納入** | **影像 T1**：判讀所需資訊一目了然。**細胞 T4**：編輯過程中若數值達留單標準，即時回饋（本區數據會即時反映）。 |
| **建議優化重點** | （1）可加「編輯細胞分類時，左側數據會即時更新；若某類別百分比超過留單閾值，會以顏色提醒，避免做完才發現需人工鏡檢」。（2）保留「留單標準依醫院設定（如林口長庚）」之說明。 |

---

### 步驟 3：右上 — 工具列

| 項目 | 內容 |
|------|------|
| **現有** | 檢視進度、縮放 50%～200%、儲存並核發報告。 |
| **設計目的可納入** | **防呆**：儲存時若未分類細胞或進度未達 100% 會阻擋並提示，對應規格中的「防止遺漏」。**T5**：縮放可減輕長時間閱片的視覺負擔（與檢體管理頁的字體縮放同一人因考量）。 |
| **建議優化重點** | （1）進度條可說明為「確保每顆細胞都被檢視過，避免漏判」。（2）「儲存並核發報告」可點出「系統會檢查是否還有未分類細胞或未檢視完成，通過後才會開啟報告核發畫面」。 |

---

### 步驟 4：右下 — 細胞區塊

| 項目 | 內容 |
|------|------|
| **現有** | 依分類群組顯示細胞圖卡。選取：單選、Ctrl 多選、Shift 範圍選、單手模式（按住右鍵＋左鍵→放開右鍵開選單）。編輯：右鍵選單或拖曳到另一群組。 |
| **設計目的可納入** | **影像 T3**：異常細胞置頂、紅色邊框與警告圖示，確保高風險細胞不被遺漏。**細胞 T1**：單手模式為減少顯微鏡與鍵盤間手部切換、減輕負擔。**細胞 T2**：群組邊框與右鍵選單分層（AI 建議＞常用＞異常＞其他）、加大按鈕，提升操作精準度與容錯。**影像 T2**：右鍵選單中整合 AI 信心與建議，輔助重分類決策。**細胞 T4**：群組標題即時顯示數量與百分比，達閾值時變色警示。 |
| **建議優化重點** | （1）先說明「需留單的異常細胞（如 Blast）會集中在最上方並以紅框標示，避免在大量正常細胞中漏看」。（2）單手模式說明可強調「適合一手在顯微鏡、一手操作滑鼠時使用，不需按鍵盤」。（3）右鍵選單可點出「會顯示 AI 建議分類與信心度，可作為修正時的參考」。（4）「群組標題的數字與百分比會即時更新，超過留單標準時會變紅提醒」。 |

---

### 步驟 5：進入報告核發

| 項目 | 內容 |
|------|------|
| **現有** | 編輯完成且無未分類細胞、檢視進度達標後，點擊「儲存並核發報告」會開啟報告核發畫面。請點擊此按鈕繼續教學。 |
| **設計目的可納入** | 防呆邏輯的總結；銜接報告核發頁。 |
| **建議優化重點** | （1）可再強調一次「系統會檢查未分類與進度，通過後才開啟報告核發，避免漏報」。（2）選填：「報告核發頁會用紅／黃／綠橫幅再次提醒留單與異常狀態」。 |

## 四、報告核發頁（REPORT_STEPS）— 逐步驟討論

### 步驟 0：報告核發 — 整體介面

| 項目 | 內容 |
|------|------|
| **現有** | 本頁用於確認數據後簽核。依左上→左下→右上→右下：左上為標題與風險、左下為 WBC 表、右上為病患資訊、右下為其他發現與 CBC。底部為簽核按鈕。 |
| **設計目的可納入** | **報告 T1**：風險橫幅主動攔截高風險並以顯眼方式警告。**報告 T2**：同一畫面並列流式、AI、人員編輯、前次報告與 CBC，解決決策支援資訊斷裂。 |
| **建議優化重點** | （1）開頭加「簽核前可在同一畫面比對流式細胞儀、AI、您編輯後的結果與前次報告，減少來回查詢」。（2）「頂部橫幅會依留單／異常／正常顯示紅／黃／綠，提醒您是否需要留單或覆核」。 |

---

### 步驟 1：左上 — 標題與風險橫幅

| 項目 | 內容 |
|------|------|
| **現有** | 檢體編號與狀態膠囊；風險橫幅依留單條件顯示紅（需留單）／黃（異常未達留單）／綠（正常）。 |
| **設計目的可納入** | **報告 T1**：當報告達留單標準時系統主動攔截並以顯眼方式警告，避免誤發高風險報告。 |
| **建議優化重點** | （1）可寫「紅色橫幅表示已達留單標準，簽核前請再次確認是否需人工鏡檢或留單；黃色為有異常但未達留單；綠色為目前無異常」。（2）強調「設計目的是在最後關頭再次提醒，降低核發風險報告的機率」。 |

---

### 步驟 2：左下 — 血球分類計數 (WBC)

| 項目 | 內容 |
|------|------|
| **現有** | 表格依細胞類型列出流式計數、AI、人員編輯、前次報告；人員編輯欄為藍底標示。 |
| **設計目的可納入** | **報告 T2**：整合分類比較表橫向並列四種數據，利於一致性比對；異常數據整列紅色＋警示圖示。 |

## 五、登入頁是否納入教學？

目前教學從「檢體管理」開始，**登入頁**沒有獨立步驟。若希望納入：

- **可補充的設計意義**：登入後集中於同一平台，與方案總述中的「集中化版本管理、單一入口」一致；若未來接 SSO／院內帳號，可一句帶過。
- **建議**：若教學對象多為院內首次使用者，可在「歡迎使用」步驟中加一句「請先以您的帳號登入，登入後會進入檢體管理頁」；不需單獨一步驟，除非您希望「第一次使用」從登入開始點擊下一步。
